# 2.1. Gerenciando Cotações (Kanban)

A plataforma Starken Insurance utiliza uma visualização no estilo **Kanban** para gerenciar o fluxo de cotações, proporcionando uma maneira intuitiva e visual de acompanhar o progresso de cada negociação. Esta abordagem permite que os usuários identifiquem rapidamente gargalos e priorizem tarefas.

#### Estrutura da Página

A página é organizada com abas e uma barra de ferramentas para facilitar a navegação e o gerenciamento.

* **Abas de Visualização:**
  * **Cotação (padrão):** Exibe o painel Kanban com todas as cotações ativas.
  * **Em andamento:** Filtro específico para cotações que estão em progresso.
  * **Declinados e Concluídos:** Uma visão separada para cotações que já foram finalizadas, seja com sucesso ou não.
* **Barra de Ferramentas:** Oferece acesso a ações importantes:
  * **Criar cotação:** Inicia o processo de uma nova cotação.
  * **Filtrar, Ordenar e Visualizar colunas:** Ferramentas para personalizar a exibição dos dados.
  * **Lista de atividades:** Acesso ao histórico de atividades relacionadas às cotações.

#### O Painel Kanban

O painel Kanban organiza as cotações em colunas que representam cada etapa do funil de vendas. Cada coluna exibe o número de cotações atualmente nesse estágio.

As etapas do fluxo são:

1. **Aguardando QAR:** Cotações que aguardam o preenchimento do Questionário de Avaliação de Risco pelo segurado ou corretor.
2. **Validação:** As informações enviadas estão sendo validadas pela equipe interna.
3. **Em análise:** A cotação foi submetida e está em análise pelas seguradoras.
4. **Negociação:** Propostas foram recebidas e estão em fase de negociação com o cliente.
5. **Assinatura:** A proposta foi aceita e aguarda a assinatura dos documentos.
6. **Em emissão:** A documentação foi assinada e a apólice está em processo de emissão.

#### Cards de Cotação

Cada cotação é representada por um "card" no painel Kanban, que resume as informações mais importantes:

| Elemento             | Descrição                                                                        |
| -------------------- | -------------------------------------------------------------------------------- |
| **Prioridade**       | Uma etiqueta colorida (ex: Baixa, Média, Alta) que indica a urgência da cotação. |
| **Origem**           | Informa a fonte da cotação (ex: Prospecção, Novo Cliente).                       |
| **Nome do Segurado** | Identificação do cliente final.                                                  |
| **ID da Cotação**    | Um número de identificação único para rastreamento.                              |
| **Tipo de Produto**  | O produto de seguro associado (ex: RC Ambiental, Transportes).                   |
| **Corretora**        | A corretora responsável pela negociação.                                         |

Os cards também possuem ícones de ação que permitem ao usuário executar tarefas específicas diretamente do painel, como editar ou visualizar detalhes adicionais.
